Exercise 1 - Cloud Service Providers

In its simplest terms, the Cloud is a vast network of devices that certain companies have made accessible to their customers via the internet, that has existed since 1996! Companies such as Yahoo and Hotmail offered their own Software as a Service models, although you may also be familiar with the capabilities of Cloud Storage technologies widely adopted in the contemporary computing age, such as OneDrive and Google Drive. That being said, it is not simply storage that can be provisioned in a Cloud environment, but instead whole networks, infrastructure, software and many more applications that businesses are adopting as a more cost-effective method of IT delivery.

Due to the many features and functionalities of Cloud environments, many different companies exist that provide these services, known as Cloud Service Providers (CSPs). The most popular Cloud Service Providers are Amazon Web Services (AWS) and Microsoft Azure. In this exercise, we will discuss different aspects of both AWS and Azure.

Learning Outcomes

After completing this exercise, you should be able to:

  • Describe Microsoft Azure DataCentres & Regions
  • Describe Microsoft Azure Pricing Calculator
  • Describe Amazon Web Services (AWS) Datacentres & Regions

Task 2 - Microsoft Azure Pricing Calculator

Creating resources on a Cloud service provider involves a specific cost. Companies need to assess the costs involved when planning to migrate resources to a Cloud Service provider. Any resource that is created will have a specific cost associated with it, and the subscription will be charged according to the consumption of the specific resource on a pay-as-you-go billing methodology. In this task, the Azure pricing calculator will be used to investigate the costs involved in the hosting of resources on the Azure platform.

Step 1

In Microsoft Edge, open a new tab and browse to the following website:

https://azure.microsoft.com/en-us/pricing/calculator

Step 2

In the Microsoft Edge browser, scroll down and select Virtual Machines.

Step 3

In Microsoft Edge, select View in the Virtual Machine added pop-up window.

Step 4

In Microsoft Edge in the Region section, change the selection to UK South.

Note: The different regions in which resources can be created in Azure are displayed. The region can be changed according to your location. For the purpose of the exercise, the UK South region will be used.

Step 5

In the Instance section change the size to the following:

D8d v4: 8 vCPUs, 32 GB RAM, 300 GB Temporary storage
Note: Different Virtual machine sizes have different costs. A company can use the pricing calculator to estimate the amount a specific virtual machine will cost per month, depending on the resource utilization.

Step 6

In Microsoft Edge, scroll to the bottom of the page to the Estimated monthly cost section.

The default currency will be US Dollar ($).

Step 7

Select United Kingdom - Pound from the drop-down.

Note: The Azure pricing calculator's currency can be changed according to a specific currency to get a better understanding of the cost involved when creating resources on the Azure platform. For this lab exercise, it will be changed to British pounds.

Step 8

Close the Microsoft Edge browser.

Note: Leave the PLABWIN10 in its current state and move on to the next task.

Task 4 - Amazon Web Services Pricing Calculator

Creating resources on a Cloud Service provider involves a specific costing structure. Specific costs are associated with different resources that are created. A company that considers migrating its resources can use a cloud service provider’s pricing calculator to determine an estimated cost for the specific resources. In this task, the Amazon Web Services pricing calculator will be used to determine the cost for a specific resource being hosted on the AWS platform.

Step 1

Ensure you are connected to . Open the Microsoft Edge browser, and browse to the following URL:

https://calculator.aws/

Step 2

In the Select your cookie preferences pop-up window, select Accept all.

Step 3

In the Microsoft Edge browser, click Create estimate.

Step 4

In the Microsoft Edge browser, scroll down to Amazon EC2 and select Configure.

Step 5

In the Region drop-down menu, select Europe (London).

Alert: This step may not be required.

Step 6

In the Confirm region change pop-up window, select Change Region.

Note: Depending on the region you reside in, it can be changed accordingly. For the purpose of the lab, the EU (London) region will be used.

Step 7

Scroll to the middle of the window without changing any parameters. You can see that Amazon is providing different costing options based on pre-defined specifications.

Note: You can change the specifications of the EC2 instance to regenerate estimate figures. Several different operating systems are available with different price ranges. A company can select the specific operating system and estimate the cost per month for the operating system.

Step 8

Close the Microsoft Edge browser.
